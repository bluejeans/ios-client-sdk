ModeratorControlsServiceProtocol

public protocol ModeratorControlsServiceProtocol

ModeratorControlsService can be used to control moderator specific actions in the meeting.

  • isModeratorControlsAvailable

    True if we are moderator and the controls are available, false if they are not. Nil out of a meeting.

    Declaration

    Swift

    var isModeratorControlsAvailable: BJNObservable<Bool?> { get }
  • isRecordingFeatureAvailable

    Observable that provides about recording feature is available for the meeting.

    Declaration

    Swift

    var isRecordingFeatureAvailable: BJNObservable<Bool?> { get }
  • recordingState

    Observable that provides current recordingState

    Declaration

    Swift

    var recordingState: BJNObservable<RecordingState?> { get }
  • startRecording(onCompletion:)

    Start to record the current meeting. Note: recording can be started and stopped by other moderators so you should rely on the recordingState for UX and not just the result of the startRecording function.

    Declaration

    Swift

    func startRecording(onCompletion: @escaping (Result<RecordingSuccess, BJNClientSDK.RecordingError>) -> Void)
  • stopRecording()

    End recording for the current meeting.

    Declaration

    Swift

    func stopRecording()

    End current meeting for all participant in timeInSeconds seconds. This will also leave the meeting for the moderator calling this function.
